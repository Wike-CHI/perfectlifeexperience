# 多导师育成津贴方案

## 场景定义

**单导师**：
```
B培养 → C升级
B是C的唯一导师
```

**多导师**：
```
A和C一起培养 → B升级
A和C都是B的导师
怎么分津贴？
```

---

## 方案1：主导师制（简单粗暴）

### 规则

- 每个下级只有**一个导师**
- 导师 = 直接推荐人
- 津贴全给推荐人

### 场景

```
A（金牌）推荐 → B（银牌）
B（银牌）推荐 → C（铜牌）
C（铜牌）推荐 → D（普通）
```

**D升级到铜牌后**：
- 导师 = C（直接推荐人）
- C拿：3元（基础佣金） + 0.06元（育成津贴）
- B拿：6元（基础佣金），没有津贴
- A拿：10元（基础佣金），没有津贴

**优点**：
- ✅ 简单明了
- ✅ 计算方便
- ✅ 避免纠纷

**缺点**：
- ❌ 不符合"多导师培养"的现实
- ❌ 间接推荐人没有动力帮忙

---

## 方案2：多导师按比例分

### 规则

- 允许多个导师
- 按"培养贡献度"分津贴
- 贡献度 = 系统自动记录（培训次数、指导时长等）

### 场景

```
A（金牌）和B（银牌）一起培养 → C（铜牌）
A贡献度：60%
B贡献度：40%
```

**C升级到银牌后，卖货赚6元佣金**：
- 育成津贴：6 × 2% = 0.12元
- A拿：0.12 × 60% = 0.072元
- B拿：0.12 × 40% = 0.048元

**问题**：
- 怎么判定"培养贡献度"？
  - 培训次数？
  - 指导时长？
  - 下级业绩增长？
- 系统复杂度太高

---

## 方案3：层级递减制（推荐）

### 规则

- **直接推荐人** = 主导师，拿100%津贴
- **间接推荐人** = 辅导导师，拿50%津贴
- 津贴从下级佣金里扣，但不超过下级佣金的2%

### 场景

```
A（金牌）推荐 → B（银牌）推荐 → C（铜牌）推荐 → D（普通）
```

**D升级到铜牌后，卖货赚3元佣金**：
- 主导师C：3 × 2% = 0.06元
- 辅导导师B：3 × 1% = 0.03元（50%津贴）
- 总津贴：0.09元（从3元里扣）

**D实际拿**：3 - 0.09 = 2.91元

**但这样会超20元池！**

## 方案4：津贴池制（简化版）

### 核心逻辑

育成津贴**不从20%佣金池出**，而是**额外给**。

### 场景

```
A（金牌）推荐 → B（银牌）推荐 → C（铜牌）推荐 → D（普通）
D卖货100元
```

**基础佣金池20元**：
- A：10元
- B：6元
- C：3元
- D：1元

**D升级到铜牌后**：
```
D卖货100元
```

**基础佣金池20元**：
- A：10元
- B：6元
- C：3元
- D：1元

**育成津贴（额外给）**：
- 主导师C：D佣金（3元）× 2% = 0.06元
- 辅导导师B：D佣金（3元）× 1% = 0.03元
- 总津贴：0.09元

**总发放**：20元（基础） + 0.09元（津贴） = 20.09元

**问题**：
- ❌ 超过20元池
- ❌ 公司拿79.91元，不是80元

## 方案5：主导师独占制（推荐）

### 核心逻辑

- **导师 = 直接推荐人**
- **育成津贴 = 下级佣金×2%**
- **津贴从下级佣金里扣**

### 场景

```
A（金牌）推荐 → B（银牌）推荐 → C（铜牌）推荐 → D（普通）
```

**D升级到铜牌后，卖货100元**：

| 人物 | 等级 | 计算方式 | 实际佣金 |
|------|------|----------|----------|
| A | 金牌 | 10元 | 10元 |
| B | 银牌 | 6元 | 6元 |
| C | 铜牌 | D的导师 | 3元 + 0.06元（津贴） |
| D | 铜牌 | 卖货人 | 3元 - 0.06元（给导师） |
| **总计** | | | **20元** ✅ |

**C升级到银牌后，卖货100元**：

| 人物 | 等级 | 计算方式 | 实际佣金 |
|------|------|----------|----------|
| A | 金牌 | 10元 | 10元 |
| B | 银牌 | C的导师 | 6元 + 0.12元（津贴） |
| C | 银牌 | 卖货人 | 6元 - 0.12元（给导师） |
| **总计** | | | **20元** ✅ |

**优点**：
- ✅ 简单明了
- ✅ 不超20元池
- ✅ 激励直接推荐人培养下级
- ✅ 间接推荐人通过基础佣金获利

**缺点**：
- ❌ 间接推荐人没有津贴（但他们已经拿了更高的基础佣金）

## 方案6：多导师分层制（备选）

### 核心逻辑

允许最多2个导师：
- **主导师**（直接推荐人）：拿1.5%
- **辅导导师**（间接推荐人）：拿0.5%
- **总津贴**：2%（从下级佣金扣）

### 场景

```
A（金牌）推荐 → B（银牌）推荐 → C（铜牌）推荐 → D（普通）
```

**D升级到铜牌后，卖货100元**：

| 人物 | 等级 | 计算方式 | 实际佣金 |
|------|------|----------|----------|
| A | 金牌 | 10元 | 10元 |
| B | 银牌 | 辅导导师 | 6元 + 0.03元（津贴） |
| C | 铜牌 | 主导师 | 3元 + 0.045元（津贴） |
| D | 铜牌 | 卖货人 | 3元 - 0.075元（给导师） |
| **总计** | | | **20元** ✅ |

**津贴分配**：
- 主导师C：3元 × 1.5% = 0.045元
- 辅导导师B：3元 × 0.5% = 0.015元
- 等等，算错了...

让我重新算：

**D赚3元佣金**：
- 主导师C：3 × 1.5% = 0.045元
- 辅导导师B：3 × 0.5% = 0.015元
- 总津贴：0.06元（3元的2%）

**D实际拿**：3 - 0.06 = 2.94元

**B实际拿**：6 + 0.015 = 6.015元
**C实际拿**：3 + 0.045 = 3.045元

**总佣金**：10 + 6.015 + 3.045 + 2.94 = 22元 ❌

**问题**：还是超了！

## 正确的计算方式

方案6应该这样算：

**D升级到铜牌后，卖货100元**：

**基础佣金池20元**分配：
- A（金牌）：10元
- B（银牌）：6元
- C（铜牌）：3元
- D（铜牌）：1元

等等，D都升级到铜牌了，为什么基础佣金还是1元？

因为推荐链是：A → B → C → D → 客户
- A是第1级（金牌）：10%
- B是第2级（银牌）：6%
- C是第3级（铜牌）：3%
- D是第4级（相对C）：1%

**所以D的基础佣金永远是1%（因为他是第4级）**，不管他自己是什么等级！

**正确理解**：
- **基础佣金**：由你在推荐链中的位置决定（第几级）
- **等级**：决定你的下级升级时，你能否拿育成津贴
