# 推广体系重新设计

## 核心约束

- 佣金池硬上限：20%
- 兼顾晋升体系和代理体系
- 激励上级帮助下级升级

---

## 方案A：按推荐链位置分佣金（推荐）

### 核心逻辑

**基础佣金 = 由你在推荐链中的位置决定，与个人等级无关**

### 规则


| 推荐链位置            | 基础佣金 | 说明       |
| --------------------- | -------- | ---------- |
| 第1级（离客户最近）   | 1%       | 直接推荐人 |
| 第2级                 | 3%       | 间接推荐人 |
| 第3级                 | 6%       | 三级推荐人 |
| 第4级（离总公司最近） | 10%      | 顶级推荐人 |
| **总计**              | **20%**  |            |

### 示例

**推荐链**：

```
金牌A → 银牌B → 铜牌C → 普通D → 客户
```

**D卖货100元，佣金分配**：

- D（第1级）：1元
- C（第2级）：3元
- B（第3级）：6元
- A（第4级）：10元
- **总计：20元** ✅

**D升级到铜牌后，卖货100元**：

- D（第1级）：1元（**位置不变，佣金不变**）
- C（第2级）：3元
- B（第3级）：6元
- A（第4级）：10元
- **总计：20元** ✅

### 等级的作用


| 等级 | 解锁权益   | 说明                                       |
| ---- | ---------- | ------------------------------------------ |
| 普通 | 无         | 基础佣金1元                                |
| 铜牌 | 复购奖3%   | 老客户复购时，额外拿3元（从20%池里扣）     |
| 银牌 | 管理奖2%   | 下级卖货时，额外拿2元（从20%池里扣）       |
| 金牌 | 育成津贴2% | 培养的下级升级后，拿下级佣金×2%（额外给） |

### 优点

✅ 20%硬上限，永不超支
✅ 下级升级不影响上级基础佣金
✅ 等级只解锁额外奖励
✅ 逻辑简单，易于理解

### 缺点

❌ D升级后，基础佣金还是1元（可能觉得不公平）

## 方案B：基础佣金固定，奖励额外给

### 核心逻辑

**基础佣金固定20%，额外奖励从公司利润出**

### 规则


| 等级 | 基础佣金 | 额外奖励   | 来源     |
| ---- | -------- | ---------- | -------- |
| 金牌 | 10%      | 育成津贴2% | 公司利润 |
| 银牌 | 6%       | 管理奖2%   | 公司利润 |
| 铜牌 | 3%       | 复购奖3%   | 公司利润 |
| 普通 | 1%       | 无         | -        |

### 示例

**D（普通）卖货100元**：

- A：10元
- B：6元
- C：3元
- D：1元
- 公司：80元
- **总计：100元** ✅

**D升级到铜牌后，卖货100元，客户是老客户**：

- A：10元
- B：6元
- C：3元
- D：1元 + 3元（复购奖）= 4元
- 公司：80 - 3 = 77元
- **总计：100元** ✅

### 优点

✅ 20%基础佣金永不超支
✅ 升级后收益明显增加（D从1元变成4元）
✅ 激励效果强

### 缺点

❌ 公司利润不确定（80元可能变成77元）
❌ 需要核算额外奖励对公司利润的影响

## 方案C：取消复杂奖励，简化体系

### 核心逻辑

**只有基础佣金，没有复购奖、管理奖、育成津贴**

### 规则


| 推荐链位置 | 基础佣金 |
| ---------- | -------- |
| 第1级      | 1%       |
| 第2级      | 3%       |
| 第3级      | 6%       |
| 第4级      | 10%      |

### 升级机制

**升级 = 推荐链位置上移**

- 普通（第1级）升级到铜牌（第2级）→ 基础佣金从1%变成3%
- 铜牌（第2级）升级到银牌（第3级）→ 基础佣金从3%变成6%
- 银牌（第3级）升级到金牌（第4级）→ 基础佣金从6%变成10%

### 示例

**初始**：

```
A(金牌10%) → B(银牌6%) → C(铜牌3%) → D(普通1%) → 客户
D卖货100元：A10 + B6 + C3 + D1 = 20元
```

**D升级到铜牌后**：

```
A(金牌10%) → B(银牌6%) → C(铜牌3%)
                          ↓
                       D(铜牌3%) → 客户
D卖货100元：A10 + B6 + C3 = 19元（C失去了D）
```

**问题**：C失去D，C不愿意培养D！

### 解决：C拿育成津贴

```
D卖货100元：A10 + B6 + C3 + D3 - 0.06(给C) = 21.94元 ❌ 超支！
```

或者：

```
D卖货100元：A10 + B6 + C(0.06津贴) + D(2.94) = 19元 ✅
```

但C收益暴跌（3元 → 0.06元）！

## 方案D：混合方案（推荐）

### 核心逻辑

**基础佣金按位置 + 等级奖励从公司利润出**

### 基础佣金（20%硬上限）


| 位置  | 佣金 |
| ----- | ---- |
| 第1级 | 1%   |
| 第2级 | 3%   |
| 第3级 | 6%   |
| 第4级 | 10%  |

### 等级奖励（额外给）


| 等级 | 奖励     | 条件       | 来源     |
| ---- | -------- | ---------- | -------- |
| 铜牌 | 复购奖   | 老客户复购 | 公司利润 |
| 银牌 | 管理奖   | 下级卖货   | 公司利润 |
| 金牌 | 育成津贴 | 下级升级   | 公司利润 |

### 示例

**D（普通）卖货100元**：

- 基础佣金：A10 + B6 + C3 + D1 = 20元
- 等级奖励：无
- 公司：80元

**D升级到铜牌后，卖货100元，老客户复购**：

- 基础佣金：A10 + B6 + C3 + D1 = 20元
- 等级奖励：D拿3元（复购奖）
- 公司：80 - 3 = 77元

**C（铜牌）培养D升级**：

- D卖货时，C拿3元基础佣金
- D升级后，C额外拿 D佣金×2% = 0.06元（育成津贴）
- C总收益：3 + 0.06 = 3.06元

### 优点

✅ 20%基础佣金硬上限
✅ 升级后收益增加
✅ 上级有动力帮助下级（C从3元变成3.06元）
✅ 逻辑清晰

### 缺点

❌ 公司利润不确定（77-80元之间浮动）
❌ 需要监控额外奖励总额
