### TDD开发流程

1. **RED - 编写失败的测试**
   ```javascript
   it('应该实现新功能', () => {
     const result = newFunction();
     assert.strictEqual(result, expected);
   });
   ```

2. **GREEN - 编写最小代码通过测试**
   ```javascript
   function newFunction() {
     return expected; // 刚好让测试通过
   }
   ```

3. **REFACTOR - 重构优化代码**
   ```javascript
   function newFunction() {
     // 在测试保护下重构
     return calculateExpected();
   }
   ```
