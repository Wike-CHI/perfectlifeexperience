## 📊 订单状态说明

### 完整状态列表（7种）

| 状态值 | 显示名称 | 颜色 | 说明 |
|--------|---------|------|------|
| `pending` | 待付款 | 橙色 | 订单已创建，等待用户支付 |
| `paid` | 已支付 | 绿色 | 用户已支付，等待发货 |
| `shipping` | 配送中 | 鼠尾草绿 | 商品配送中 |
| `completed` | 已完成 | 琥珀金 | 订单完成 |
| `refunding` | 退款中 | 橙色 | 用户申请退款 |
| `refunded` | 已退款 | 棕灰色 | 退款完成 |
| `cancelled` | 已取消 | 红色 | 订单已取消 |

### 状态分组（简化显示）

为了便于管理，系统将7种状态简化为5个分组：

| 分组名称 | 包含的状态 | 用途 |
|---------|-----------|------|
| 全部 | 所有状态 | 查看所有订单 |
| 待付款 | `pending` | 需要催付的订单 |
| 处理中 | `paid`, `shipping` | 需要发货或配送中的订单 |
| 已完成 | `completed` | 历史订单 |
| 退款 | `refunding`, `refunded` | 需要处理的退款 |
| 已取消 | `cancelled` | 已取消的订单 |

## 👤 用户端流程

### 1. 用户下单

**操作步骤**:
1. 浏览商品，选择规格
2. 点击"加入购物车"
3. 进入购物车，确认订单
4. 填写收货地址
5. 选择支付方式（微信支付/余额支付）
6. 确认下单

**订单创建**:
- 订单状态：`pending`（待付款）
- 系统生成订单号（格式：ORD + 时间戳）

### 2. 用户支付

**操作方式**:
- 在订单详情页点击"立即支付"
- 调用微信支付或余额支付

**支付成功后**:
- 订单状态：`pending` → `paid`（已支付）
- 用户可查看订单详情

### 3. 用户查看订单

**查看入口**:
- 小程序底部菜单 → "订单"
- 查看所有订单列表

**订单列表功能**:
- 按状态筛选订单（待付款、处理中、已完成、退款、已取消）
- 查看订单号、金额、状态
- 点击订单查看详情

**订单详情包含**:
- 订单号、下单时间、支付时间
- 商品列表（图片、名称、价格、数量）
- 收货地址信息
- 订单状态和操作按钮

### 4. 用户操作按钮

根据订单状态，用户可以执行不同操作：

| 订单状态 | 可执行操作 |
|---------|-----------|
| `pending`（待付款） | • 取消订单<br>• 立即支付 |
| `shipping`（配送中） | • 确认收货 |
| `completed`（已完成） | • 再次购买 |

### 5. 用户申请退款

**申请条件**:
- 订单状态为 `paid`（已支付）或 `shipping`（配送中）

**操作步骤**:
1. 进入订单详情
2. 点击"申请退款"
3. 选择退款商品和数量
4. 填写退款原因
5. 上传凭证图片（可选）
6. 提交申请

**退款流程**:
- 订单状态：`paid/shipping` → `refunding`（退款中）
- 等待管理员审核

## 👨‍💼 管理员端流程

### 1. 登录管理后台

**登录入口**:
- 小程序管理入口（隐蔽式）
- 输入管理员账号和密码
- 验证通过后进入管理后台

### 2. 订单管理首页

**功能概览**:
```
┌─────────────────────────────────────┐
│  [全部] [待付款] [处理中] [已完成]    │  ← 状态筛选
│  [退款] [已取消]                      │
├─────────────────────────────────────┤
│  🔍 搜索订单号                       │  ← 搜索功能
├─────────────────────────────────────┤
│  ┌───────────────────────────────┐  │
│  │ 订单卡片 1                    │  │  ← 订单列表
│  │ • 订单号、状态                │  │
│  │ • 商品图片（前5个）          │  │
│  │ • 金额、时间                  │  │
│  │ • [详情] [更新状态] [删除]    │  │  ← 操作按钮
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │ 订单卡片 2                    │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 3. 筛选订单

**状态筛选**:
- 点击顶部标签切换不同状态
- 标签显示该状态订单数量

**搜索订单**:
- 输入订单号搜索
- 实时搜索结果

### 4. 订单操作

管理员对每个订单可以执行以下操作：

#### 操作1：查看详情

**操作方式**:
- 点击订单卡片
- 或点击"详情"按钮

**详情页显示**:
- 订单状态卡片（状态、图标、说明）
- 订单基本信息（订单号、时间、金额）
- 用户信息（头像、昵称、手机号）
- 收货地址（含地图预览）
- 商品清单（图片、名称、规格、数量、价格）
- 操作按钮

#### 操作2：更新订单状态

**操作方式**:
- 在订单列表点击"更新状态"
- 或在订单详情页点击"更新状态"

**状态选择**:
系统弹出状态列表，管理员选择新状态：
```
• 待付款 (pending)
• 待发货 (paid)
• 待收货 (shipping)
• 已完成 (completed)
• 退款中 (refunding)
• 已退款 (refunded)
• 已取消 (cancelled)
```

**更新后**:
- 订单状态立即更新
- 用户端订单列表同步显示新状态
- 系统记录状态变更日志

#### 操作3：删除订单

**操作方式**:
- 在订单列表点击"删除"按钮
- 或在订单详情页点击"删除订单"

**安全提示**:
- 系统弹出二次确认对话框
- 提示"删除后无法恢复，确定要删除该订单吗？"
- 确认后才执行删除

**删除后**:
- 订单从列表中移除
- 数据库中标记为已删除（软删除）
- 用户端不再显示该订单

### 5. 退款管理

**退款入口**:
- 管理后台 → 退款管理
- 查看所有退款申请

**退款状态**:
| 状态 | 说明 |
|------|------|
| 待审核 | 等待管理员审核 |
| 已批准 | 同意退款申请 |
| 已拒绝 | 拒绝退款申请 |
| 退款中 | 退款处理中 |
| 已完成 | 退款完成 |

**退款操作**:
1. 查看退款详情
2. 审核退款申请（批准/拒绝）
3. 填写审核意见
4. 处理退款

## 🔄 订单状态流转

### 正常订单流程

```
用户下单
   ↓
pending（待付款）
   ↓
用户支付
   ↓
paid（已支付）
   ↓
管理员发货
   ↓
shipping（配送中）
   ↓
用户确认收货
   ↓
completed（已完成）
```

### 退款流程

```
用户申请退款
   ↓
订单状态变为 refunding（退款中）
   ↓
管理员审核
   ├─ 批准 → refunded（已退款）
   └─ 拒绝 → 恢复原状态（paid/shipping）
```

### 取消订单流程

```
用户取消订单（pending状态）
   ↓
cancelled（已取消）
```

## ❓ 常见问题

### Q1: 用户如何知道订单状态更新？

**答**: 用户可以通过以下方式查看：
1. **订单列表**: 小程序 → 订单 → 查看所有订单及状态
2. **订单详情**: 点击订单查看详细信息
3. **状态颜色**: 不同状态用不同颜色标识
   - 橙色：待付款、退款中
   - 绿色：已支付
   - 鼠尾草绿：配送中
   - 琥珀金：已完成
   - 红色：已取消

### Q2: 管理员更新状态后，用户端会立即看到吗？

**答**: 是的，订单状态实时同步：
- 管理员更新状态 → 数据库更新
- 用户刷新订单列表 → 看到最新状态
- 无需等待，无延迟

### Q3: 订单状态可以逆向修改吗？

**答**: 可以，但需要谨慎：
- 管理员可以将订单改为任意状态
- 例如：已完成 → 退款中
- 但建议按照正常流程操作

### Q4: 删除订单后能恢复吗？

**答**: 不能，删除是永久性操作：
- 系统会二次确认
- 提示"删除后无法恢复"
- 建议先导出数据再删除

### Q5: 如何处理退款申请？

**答**: 退款处理流程：
1. 进入"退款"状态标签
2. 查看退款申请详情
3. 审核退款理由和凭证
4. 选择批准或拒绝
5. 填写审核意见
6. 提交审核结果

### Q6: 订单号格式是什么？

**答**: 订单号格式：`ORD` + 时间戳 + 随机码
- 示例：`ORD1773045444892DXBRZO`
- 唯一标识每个订单
- 可用于搜索和查询

### Q7: 配送范围是多少？

**答**: 当前配送范围：10公里
- 门店位置：瑞安市
- 超出10公里显示"超出配送范围"
- 在10公里内显示"在配送范围内"

### Q8: 用户可以修改订单吗？

**答**: 下单后不能修改，只能：
- 取消订单（待付款状态）
- 申请退款（已支付/配送中）
- 确认收货（配送中）

## 🎯 最佳实践

### 管理员建议

1. **定期检查待付款订单**: 及时联系未支付用户
2. **快速处理退款**: 提高用户满意度
3. **准确更新状态**: 让用户了解订单进度
4. **谨慎删除订单**: 删除前确认无误

### 用户建议

1. **及时支付**: 待付款订单24小时后可能取消
2. **确认收货**: 收到商品后及时确认
3. **申请退款**: 如有问题尽早申请退款
4. **保留凭证**: 保留订单号和支付凭证
