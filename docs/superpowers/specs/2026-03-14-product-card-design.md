# 订单详情页 - 商品卡片设计文档

**日期**: 2026-03-14
**作者**: Claude Code
**状态**: 设计阶段

## 项目背景

订单详情页的商品卡片需要在保持极简现代风格的同时，展示完整的商品信息。当前极简版只显示基本信息，用户需要查看更多详情（如英文名、描述、技术参数等）。

## 设计目标

1. **信息完整性**: 展示所有关键商品信息（名称、英文名、描述、参数、标签等）
2. **视觉简洁性**: 保持极简现代风格，避免过度设计
3. **用户体验**: 符合移动端浏览习惯，信息层级清晰
4. **性能考虑**: 利用已加载的商品详情数据（`productDetails` Map）

## 设计方案：垂直卡片布局

### 整体布局

```
┌─────────────────────────────┐
│  [图片: 280rpx高]           │
│                             │
├─────────────────────────────┤
│  产品名称 (32rpx, 粗体)      │
│  English Name (24rpx, 灰色)  │
│                             │
│  🔥热销  🆕新品              │
│                             │
│  产品描述文本...             │
│  (最多2行, 超出省略)         │
│                             │
│  规格: 330ml 罐装            │
│  酒厂: 精酿啤酒厂            │
│  酒精度: 5.2% | 容量: 330ml  │
│                             │
│  ────────────────────────── │
│                             │
│  ¥28.00  x2  |  ¥56.00      │
└─────────────────────────────┘
```

### 卡片结构

1. **顶部图片** (280rpx高)
   - 宽度100%, 圆角16rpx顶部
   - mode="aspectFill"保持比例

2. **信息区域** (24rpx内边距)
   - 名称组: 中文名 + 英文名
   - 标签: 热销、新品徽章
   - 描述: 最多2行，超出省略
   - 技术参数: 规格、酒厂、酒精度、容量
   - 价格区域: 单价、数量、总价

## 数据源

商品卡片从两个数据源获取信息：

1. **订单数据** (`order.products`)
   - name, price, quantity, image, specs
   - productId (用于查询详情)

2. **商品详情** (`productDetails` Map)
   - enName, description, originalPrice
   - tags, isHot, isNew
   - brewery, alcoholContent, volume
   - 通过 `getProductDetail(item.productId)` 获取

## 技术实现要点

### 条件渲染
- 英文名、描述、原价等字段使用 `v-if` 判断存在性
- 标签徽章根据 `isHot`、`isNew` 显示
- 复杂参数（酒精度|容量）使用条件组合

### 价格计算
- 单项总价: `item.price * item.quantity`
- 原价显示: 仅当 `originalPrice > price` 时显示

### 文本截断
- 描述使用CSS限制最多2行:
  ```scss
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
  overflow: hidden;
  ```

### 响应式布局
- 图片固定高度，宽度自适应
- 信息区域使用flex布局
- 价格区域左右分布

## 与现有设计的兼容性

- 保持与订单详情页其他卡片的视觉一致性
- 复用现有的颜色系统（#1A1A1A主色, #9B8B7F次要色）
- 遵循现有的间距规范（24rpx基础间距）
- 使用纯色背景，避免渐变

## 用户交互

- 卡片整体无点击跳转（订单页静态展示）
- 支持长按复制商品名称
- 价格文本可选择（便于截图分享）

## 后续优化建议

1. **性能优化**: 如果订单商品很多，考虑虚拟滚动
2. **SKU展示**: 如果商品有多个SKU，显示当前购买规格
3. **库存提示**: 如果库存不足，显示警告信息
4. **相似推荐**: 在商品卡片下方显示相似商品推荐
